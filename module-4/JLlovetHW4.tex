\noindent\textbf{Problem 2:}

Write an algorithm to reverse a singly linked list, so that the last element become the first and so on. Do NOT use Deletion - rearrange the pointers.

This algorithm takes linear time, reversing the list in a single pass. Previously I attempted to solve this recursively, using the pattern that was demonstrated in \cite{zybooks-ds2026} for traversing a linked list in reverse order through intentional ordering of recursive calls, but actually modifying the list started to become fairly involved with that strategy - not to mention that it used a fairly significant amount of space. So, I opted for this simpler option, where I have a sliding window of attention on the list. After handling a few special cases, there is a sliding window of three nodes at play, and we change the direction of the current node's next pointer to go back to the previous node before we slide the attention frame forward one node. This is performed in a while loop until we reach the boundary case represented by the end of the list, after which we can perform the reassignment of the head of the list to what was previously the tail, and the reversal is complete.

\medskip
\noindent\textbf{Answer:}

\begin{mdframed}
  \begin{verbatim}
function reverse(list: LinkedList) {
    # special cases: empty, single element
    
    # empty
    if list.head == null {
        return list
    }

    previous = list.head # initialize
    
    # single element (no reversal required)
    if previous.next == null {
        return list
    } 
    
    # standard cases from here

    current = previous.next # start incrementing
    previous.next = null # reverse pointer for head (special case)
    
    # body
    while current.next != null {   
        next = current.next # store next node
        current.next = previous # reverse pointer
        previous = current # increment forward
        current = next # increment forward
    }

    # tail
    list.head = current # set head to last node
    current.next = previous # reverse pointer
    
    # all done!
    return list
}
\end{verbatim}
\end{mdframed}

\noindent\textbf{Problem 4:}

Write a routine to interchange the mth and nth elements of a singly-linked list. You may assume that the ranks m and n are passed in as parameters. Allow for all the ways that m and n can occur. You must rearrange the pointers, not simply swap the contents.

function swap(seq: LinkedList, m, n: int) {
    # degenerate cases
    if seq.head == null or seq.head.next == null {
        return
    }
    if m > (seq.length - 1) or n > (seq.length - 1) or m == n {
        return # Error or Exception
    }
    # legitimate cases
    if m > n {
        a = n
        b = m
    }
    if n > m {
        a = m
        b = n
    }
    # From here we know that b > a by the above two blocks
    a_current = seq.head
    b_current = seq.head
    a_previous = a_current
    b_previous = b_current
    a_pos = 0 # need these to know when to stop iterating
    b_pos = 0 # need these to know when to stop iterating
    while (b_previous.next != null) and (b_pos < b) {
        if a_pos < a { # we know that a_current.next is not null
            a_previous = a_current
            a_current = a_current.next
            a_pos += 1
        }
        b_previous = b_current
        b_current = b_current.next
        b_pos += 1
    }
    # special case where a == 0
    if a == 0 {
        tmp = b_current.next
        b_current.next = a_current.next
        b_previous.next = a_current
        a_current.next = tmp
        # Note: here we are NOT setting a_previous.next
        # because in this case it is the head of the list
    }
    # all other remaining cases
    if a > 0 {
        tmp = b_current.next
        b_current.next = a_current.next
        b_previous.next = a_current
        a_previous.next = b_current
        a_current.next = tmp
    }
}
\end{verbatim}
